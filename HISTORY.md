# AI 에이전트 플랫폼 - 개발 이력서 (HISTORY.md)

이 문서는 AI 에이전트 플랫폼의 주요 변경 사항 및 업데이트 이력을 기록합니다.

## [v2.5.0] - 2026-02-22 (현재)
### 추가 및 개선 사항
- **에이전트 모드 동기화 (Mode-Aware Reasoning)**: 
    - 선택된 에이전트(`DataAnalyzer`, `ReportCreator` 등)에 따라 백엔드 프롬프트가 동적으로 변경되도록 개선.
    - **ReportCreator**: 조사된 내용을 구조화된 전문 마크다운 보고서로 생성.

> [!IMPORTANT]
> **선택 후 질문(Select-Then-Ask) 권장**: 왼쪽 메뉴에서 먼저 에이전트를 클릭하면 해당 모드에 맞는 질문 템플릿이 자동 생성됩니다. 이를 사용하면 가장 정확한 결과물을 얻을 수 있습니다.
    - `ReportCreator` 선택 시 보고서 형식 강제, `DataAnalyzer` 선택 시 인사이트 위주 답변 강제.
- **로버스트 검색 시스템 (Robust Search)**:
    - 검색 결과가 없을 시 스스로 쿼리를 수정하여 최대 3회 재시도하는 루프 구현.
    - 검색 엔진 메타데이터(`ENGINE: ...`) 표시 기능 추가.
- **투명성 강화**: 모든 답변에 실제 원본 기사 URL 링크(`Source`) 포함 의무화.
- **한국어 응답 최적화**: 어떤 소스에서 정보를 가져오든 최종 답변은 반드시 한국어로 출력되도록 가드레일 강화.
- **UI 인터랙션 개선**: 왼쪽 사이드바의 에이전트 메뉴에 선택(Glow) 효과 및 클릭 시 자동 프롬프트 입력 기능 추가.
- **사용자 인터페이스(UI) 전면 한글화**: 사이드바, 헤더, 버튼, 통계 바 및 채팅 플레이스홀더를 한국어로 전면 개편하여 완벽한 사용자 경험 제공.
- **최종 검증 완료 (10-Test Suite)**: BTS 뉴스, 스포츠 분석, 코드 생성 등 10개 핵심 시나리오에 대해 100% PASS 기록 및 안정성 확보.
- **백엔드 프롬프트 구조 최적화**: `f-string` 사전 치환과 LangChain 변수 이스케이프(`{{}}`)를 조합하여 동적 프롬프트 생성 시 발생하던 500 오류 최종 해결.
- **프롬프트 변수 이중 이스케이프 수정**: `agent_name` 변수가 불필요하게 이중 중괄호로 감싸져 발생하던 런타임 변수 누락 오류 해결.
- **실시간 이력 관리 체계 구축**: 모든 수정 및 가이드 수정 사항을 `HISTORY.md`에 실시간 기록하는 상시 업데이트 체계 마련.
- **사용설명서 및 이력서 한글화**: `MANUAL.md`와 `HISTORY.md`를 한국어로 전면 개편.
- **BTS 광화문 공연 할루시네이션 수정**: 2026년 3월 예정된 실제 공연 정보를 정확히 찾지 못하거나 가짜 링크를 생성하던 문제 해결.
- **배구 종목 혼동 수정**: 여자배구 요청 시 남자배구 팀 결과가 섞여 나오던 필터링 오류 수정.
- **백엔드 Status 500 오류 수정**: 동적 프롬프트 생성 시 발생하던 변수 미정의 및 타입 오류 해결.

## [v2.4.0] - 2026-02-21
- **뉴스 엔진 교체**: 불안정한 DuckDuckGo 뉴스 대신 구글 뉴스 RSS 피드(BeautifulSoup 활용)를 주 엔진으로 채택하여 안정성 확보.
- **의존성 안정화**: `duckduckgo-search` 라이브러리의 `timedelta` 버그를 피하기 위해 안정적인 v4.4.3으로 고정.
- **ReAct 루프 최적화**: 파싱 에러 방지를 위한 프롬프트 구조 표준화.
